1. ОБЩЕЕ ОПИСАНИЕ ПРОЕКТА 
1.1. Цель проекта 
Разработка Telegram-бота для автоматической продажи VPN-подписок с интеграцией 
панели управления VPN, системой оплаты, реферальной программой, административной 
панелью и аналитикой. 
1.2. Целевой функционал (на основе аналога iMP VPN) 
● Автоматическая выдача VPN-конфигураций после оплаты 
● Поддержка нескольких VPN-протоколов 
● Система подписок с разными тарифами (1/3/6/12 месяцев) 
● Реферальная система с бонусами 
● Автоматические уведомления об истечении подписки 
● Административная панель для управления 
● Аналитика и метрики 
● Подарочные подписки 
2. АРХИТЕКТУРА И СТЕК ТЕХНОЛОГИЙ 
2.1. Технологический стек 
Backend: 
● Python 3.11+ 
● aiogram 3.x (Telegram Bot API) 
● FastAPI (для админ-панели и webhook) 
● SQLAlchemy 2.0 (ORM) 
● asyncpg (асинхронный драйвер PostgreSQL) 
● Redis (кэш, FSM, очереди) 
● APScheduler (фоновые задачи) 
● Celery (опционально, для масштабирования) 
Панель управления VPN: 
● Marzban (рекомендуется) или 3x-ui 
● REST API для управления пользователями 
● Поддержка VLESS + Reality, VMess, Trojan, Shadowsocks 
База данных: 
● PostgreSQL 15+ 
● Резервное копирование (pg_dump + S3/облако) 
Инфраструктура: 
● Docker + Docker Compose 
● Nginx (reverse proxy, SSL) 
● Cloudflare (CDN, защита от DDoS) 
2.2. Владение кодом 
Весь исходный код передаётся заказчику полностью. Репозиторий на GitHub/GitLab с 
документацией по развёртыванию. 
3. ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ 
3.1. Пользовательский интерфейс (Telegram Bot) 
3.1.1. Главное меню 
● Кнопки: «Купить подписку», «Мой профиль», «Подарить подписку», «Реферальная 
система», «Помощь», «Меню» 
3.1.2. Покупка подписки 
● Выбор тарифа: 1 мес / 3 мес / 6 мес / 12 мес 
● Отображение цен со скидками 
● Выбор способа оплаты: СБП, Банковская карта, Криптовалюта 
● Генерация QR-кода для оплаты (СБП) 
● Ссылка на оплату (ЮKassa/CloudPayments/CryptoBot) 
● Автоматическая выдача конфигурации после подтверждения оплаты 
3.1.3. Профиль пользователя 
● Текущий тариф и дата окончания 
● Оставшиеся дни подписки 
● Количество устройств (до 5) 
● Кнопка «Продлить подписку» 
● История платежей 
3.1.4. Подарочная подписка 
● Выбор получателя из контактов Telegram 
● Выбор тарифа для подарка 
● Оплата и автоматическая выдача конфигурации получателю 
3.1.5. Реферальная система 
● Персональная реферальная ссылка: https://t.me/YourBot?start=REF_ID 
● Отображение статистики: приглашено, активных, получено дней 
● Бонусная система: 
○ За регистрацию друга: друг +3 дня, пригласивший +1 день 
○ За покупку друга: 
■ 1 мес → +10 дней 
■ 3 мес → +25 дней 
■ 6 мес → +45 дней 
■ 12 мес → +65 дней 
3.2. Административная панель (Web) 
3.2.1. Авторизация 
● HTTP Basic Auth или Telegram OAuth 
● Доступ только для администраторов (список в .env) 
3.2.2. Дашборд 
● Общая статистика: пользователи, выручка, активные подписки 
● Графики: регистрации, платежи, рост за период 
● Топ рефералов 
● Нагрузка по серверам 
3.2.3. Управление пользователями 
● Список пользователей (поиск, фильтры) 
● Просмотр профиля: подписка, история, конфиг 
● Бан/разбан пользователя 
● Ручное продление/изменение тарифа 
● Отправка сообщения пользователю 
3.2.4. Управление серверами 
● Добавление нового сервера (node) 
● Мониторинг нагрузки (CPU, RAM, трафик) 
● Включение/выключение сервера для новых пользователей 
● Распределение пользователей по серверам 
3.2.5. Управление тарифами 
● CRUD тарифов (название, цена, скидка, длительность) 
● Активация/деактивация тарифа 
● Специальные акции (промокоды) 
3.2.6. История платежей 
● Список всех транзакций 
● Фильтры по статусу, дате, способу оплаты 
● Возможность возврата 
3.2.7. Рассылки 
● Массовая рассылка всем пользователям 
● Рассылка по фильтрам (активные/истёкшие/по серверу) 
● Предпросмотр сообщения

6. АВТОМАТИЗАЦИЯ И УВЕДОМЛЕНИЯ 
6.1. Автоматические уведомления 
● За 7 дней до окончания: «Подписка заканчивается через 7 дней. Продлить?» 
● За 3 дня: «Осталось 3 дня. Не забудь продлить!» 
● За 1 день: «Завтра отключение! Срочно продли подписку.» 
● После отключения: «Подписка истекла. Восстановить доступ?» 
6.2. Периодические задачи (APScheduler) 
● Проверка истекающих подписок (каждые 6 часов) 
● Деактивация истёкших подписок в Marzban 
● Обновление статистики серверов 
● Резервное копирование БД (ежедневно) 
6.3. Обработка платежей 
● Webhook от платёжной системы → проверка подписи → обновление статуса → 
выдача подписки 
● Idempotency (защита от дублирования) 
● Retry логика при ошибках API Marzban 
7. БЕЗОПАСНОСТЬ 
7.1. Защита API Marzban 
● JWT-токен хранится в .env (не в коде) 
● Доступ к API только с IP сервера бота 
● HTTPS обязательно 
● Возможность отзыва токена 
7.2. Защита платёжных ключей 
● Все ключи в .env 
● Разделение тестовых и боевых ключей 
● Логирование без чувствительных данных 
7.3. Резервное копирование 
● Ежедневный бэкап PostgreSQL (pg_dump) 
● Хранение в S3 / Telegram канал / облако 
● Возможность восстановления за 15 минут 
● Бэкап конфигов Marzban через API 
7.4. Сценарий взлома сервера 
● База данных: только Telegram ID и метаданные (не критично) 
● API Marzban: отзыв токена, перевод на резервный сервер 
● Платёжные ключи: отзыв через личный кабинет провайдера 
● Быстрое восстановление из бэкапа на новом сервере 
8. МАСШТАБИРОВАНИЕ 
8.1. Текущая архитектура (до 5000 пользователей) 
● Один сервер: бот + БД + Redis + Marzban 
● Docker Compose 
● Nginx + Cloudflare 
8.2. Масштабирование до 10 000+ пользователей 
Узкие места и решения: 
1. VPN-сервер (один node) 
Решение: Marzban nodes — добавление серверов без остановки 
2. База данных (нагрузка на один инстанс) 
Решение: Выделенный PostgreSQL сервер, репликация read-only для аналитики 
3. Бот (один процесс) 
Решение: Docker Swarm / Kubernetes, 2-3 реплики бота 
4. Платёжные webhook'и (одновременные запросы) 
Решение: Redis очередь + Celery workers 
5. Рассылки (лимит Telegram API: 30 msg/sec) 
Решение: Rate limiting, очередь, отправка пачками 
6. Redis (один инстанс) 
Решение: Redis Sentinel / Redis Cluster 
Архитектура под 10к+: 
● Load Balancer (Nginx / HAProxy) 
● 2-3 реплики бота (Docker Swarm) 
● Выделенный PostgreSQL (Master + Replica) 
● Redis Cluster 
● 5-10 VPN-серверов (nodes) в разных локациях 
● CDN Cloudflare 
● Мониторинг: Prometheus + Grafana

10. ПЛАН РАЗРАБОТКИ 
Этап 1: Инфраструктура (День 1-2) 
● Аренда сервера (Aeza / PQ.Hosting / VDSina) 
● Установка Docker, Docker Compose 
● Поднятие PostgreSQL + Redis 
● Установка Marzban 
● Настройка домена + Cloudflare 
● Создание бота в @BotFather

● Инициализация проекта Python + aiogram 
● Подключение к БД (SQLAlchemy + asyncpg) 
● FSM (Finite State Machine) для диалогов 
● Главное меню и навигация 
● Интеграция с Marzban API (создание пользователя, получение конфига) 
Этап 3: Платежи (День 6-8) 
● Подключение ЮKassa / CryptoBot 
● Создание платежа и обработка webhook 
● Автоматическая выдача подписки после оплаты 
● Генерация QR-кода и ссылки vless:// 
Этап 4: Рефералы и уведомления (День 9-11) 
● Реферальная система (ссылки, бонусы, статистика) 
● APScheduler: проверка подписок 
● Уведомления (7, 3, 1 день, после отключения) 
Этап 5: Админ-панель (День 12-14) 
● FastAPI + шаблоны 
● Авторизация 
● Дашборд, пользователи, серверы, тарифы, платежи 
● Рассылки 
Этап 6: Тестирование и деплой (День 15-16) 
● Тестирование всех сценариев

● Проверка VPN-соединения

Этап 7: Запуск (День 17-18)
